## Decision
www/ was previously git-ignored and regenerated by `prepare-www.js` at build time, which required isotope-code to be checked out at `../isotope-code`. User requested this be removed as a dependency risk.

## What was done
1. Cloned isotope-code to `/home/runner/workspace/isotope-code`
2. Ran `prepare-www.js` with REPO_DIR=./isotope-code to produce www/ (57MB)
3. Ran `apply-android-patches.js` on www/ (72 patches applied)
4. Removed `www/` from .gitignore
5. Committed www/ to isotope-apk main

## Contents of www/ (57MB)
- `sounds/` — 42MB (crickets.wav 11.4MB, rain.wav 14.1MB, wind.wav 16.3MB)
- `assets/` — 11MB (154 JS chunks)
- `fonts/` — 360KB
- `bg.mp4` — 4.1MB
- Bridge files: android-bridge.js, android-floating-timer-bridge.js, auth-bridge.js, restore-and-launch.js, focus-bg-import.js, etc.

**Why:** Prevents build failure if user deletes isotope-code repo. CI workflow still checks out isotope-code at ISOTOPE_CODE_REF for fresh builds, but local/fallback builds work from committed www/.

**Important:** www/ is regenerated on every CI build. The committed version is only a fallback. To update it locally: `npm run prepare-www && npm run apply-patches`
