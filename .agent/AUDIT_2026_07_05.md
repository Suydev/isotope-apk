## Architecture Facts (do not re-derive)

- `src/App.tsx` in `isotope-code` is a placeholder. Real UI = `public/assets/*.js` (154 chunks).
- Frontend uses direct Supabase table operations for groups — does NOT call `create_community_group` RPC from JS yet.
- `accept_invite` IS called as an RPC from frontend: `n.rpc("accept_invite", { p_code: t })`.
- `android-bridge.js` intercepts `window.fetch` for `/__auth/*` and `/__supa/*`.
- Patches run twice in CI: before `cap sync` and after (cap sync can regenerate native files).
- `member_count` column on `groups` is denormalized and must be kept in sync — trigger added.
