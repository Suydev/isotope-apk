2. ENVIRONMENT VARIABLES
------------------------
- SUPABASE_URL (Required): No fallback. App fails if missing.
- SUPABASE_ANON_KEY (Required): Public key for user-level RLS requests.
- SUPABASE_SERVICE_ROLE_KEY (Optional): Enables ADMIN_MODE_READY. Without it, /__admin routes return 401/503.
- ADMIN_COOKIE_SECRET: Fallback to 'isotope-secret-change-me'. Risk: Default value allows trivial cookie forging.
- ADMIN_SECRET: Used for manual admin login override.
- ADMIN_EMAIL / ADMIN_PASSWORD: Used by startup backfills to seed the primary admin account.
- GEMINI_API_KEY / GROQ_API_KEY: Optional. Activates AI store patching if present.
- VALIDATION (lines 8380-8392): prints console warnings but does not halt server (soft-fail).

8. HTTP ROUTES
--------------
- OPTIONS /*: Returns CORS headers (Origin: *, Methods: GET,POST,etc).
- /__admin/login (GET): Serves admin login HTML.
- /__admin/login (POST): Sets `admin_session` cookie if password matches ADMIN_SECRET.
- /__admin/browser-proof*: Serves diagnostic UI for manual browser-to-cloud verification.
- /api/ai-config (GET): Returns {gemini:bool, groq:bool}.
- /api/auth/*:
  - signup (POST): Proxy to Supabase. Seeding happens in `bootstrapUserRows`.
  - signin (POST): Proxy to Supabase.
  - update-profile (POST/PATCH): Merges into `user_profiles.profile_data`.
  - me (GET): Returns user row.
- /api/sync/*:

  - restore (GET): requireUserAuth. Fetches state from bucket.
- /api/ai/chat (POST): requireUserAuth. Proxies to AI provider with server-side keys.
- /__admin/apply-sql (POST): Guard: `isAdminAuthed`. Proxies raw SQL to api.supabase.com using user-provided PAT.
- Static Files: Path normalization (line 8184) prevents directory traversal.
- SPA Fallback: Unknown GET requests return index.html (line 8231).
