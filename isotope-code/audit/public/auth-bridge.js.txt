PURPOSE
Acts as a bridge between the frontend and the local server for authentication purposes, specifically handling the transfer of Supabase session data to the local server's session storage.

ROLE IN THE APPLICATION
- Subsystem: Authentication / Security
- Runtime stage: Onboarding / Auth flow
- Active/legacy: Active

REFERENCES AND CONSUMERS

INTERNAL STRUCTURE
- function syncAuthToLocal(session): Posts session data to /api/auth/sync.
- window.IsotopeAuthBridge: Public API.

INPUTS
- Supabase session object (containing access_token, refresh_token).
- localStorage (to check existing tokens).

OUTPUTS AND SIDE EFFECTS
- Fetch POST requests to /api/auth/sync.
- Sets local server cookies (via API response).

DATA FLOW
1. Frontend receives Supabase session.
2. Bridge intercepts/receives session.
3. Bridge posts session to local server.
4. Local server validates and sets session cookie.
