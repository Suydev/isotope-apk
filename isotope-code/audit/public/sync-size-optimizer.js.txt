PURPOSE
Optimizes data synchronization by making intelligent decisions based on payload size and compression.

ROLE IN THE APPLICATION
- Subsystem: Data Sync
- Runtime stage: Sync process
- Active/legacy: Active

REFERENCES AND CONSUMERS
- Used by sync modules to decide whether to upload/download/merge.

INTERNAL STRUCTURE
- SYNC_SIZE_CONFIG: Thresholds (1MB large file, 50MB max upload).
- decideSyncStrategy(cloud, local): Core decision logic.
- compressForUpload(data): Uses CompressionStream (gzip).
- decompressDownload(blob): Uses DecompressionStream.

INPUTS
- Local and Cloud data snapshots.

OUTPUTS AND SIDE EFFECTS
- Sync strategy recommendation.
- Compressed/Decompressed data blobs.

DATA FLOW
1. Receives data from both sources.
2. Compares sizes and ratios.
3. Returns 'upload', 'download', 'merge', or 'skip'.
4. Performs compression if strategy is 'upload' and size is large.

SECURITY AND PRIVACY
- Logic only; ensures data size limits are respected.

OFFLINE AND LOCAL-FIRST BEHAVIOUR
- Vital for mobile/slow networks to prevent large accidental syncs.

PERFORMANCE
- Significant benefits by avoiding redundant transfers and using native compression.

FINDINGS

POSITIVE OBSERVATIONS
- Intelligent strategy based on size ratios, not just timestamps.

Advanced and efficient sync optimizer.
